## 코드 변경 요약

### 1) 적재 큐 SQL 버그 수정
- 파일: `scripts/dw_load_utils.py`
- 변경:
  - `upsert_api_retry_queue()`의 `INSERT` 컬럼 수(13개)와 값 수 불일치 문제 수정.
  - `next_retry_at`, `updated_at` 값을 row에 포함하도록 보정.
- 효과:
  - `INSERT has more target columns than expressions` 오류 제거.

### 2) OCID 수집/적재 분리
- 파일: `scripts/load_ocid.py`
- 추가/변경:
  - `collect_user_ocid_data()` 추가: API 수집 전용.
  - `load_user_ocid_payload()` 추가: DB 적재 전용.
  - `write_ocid_payload()` / `read_ocid_payload()` 추가: 페이로드 파일 저장/복원.
  - `create_user_ocid_table()`은 레거시 호환 래퍼로 유지(수집+적재 호출).
- 데이터 전달:
  - `data_json/_airflow_payloads/ocid_collect_<date>_<run_id>.json`

### 3) Character Info 수집/적재 분리
- 파일: `scripts/load_character_info.py`
- 추가/변경:
  - `collect_character_info_data()` 추가: 엔드포인트 수집 전용.
  - `load_character_info_payload()` 추가: DB upsert 전용.
  - `write_character_info_payload()` / `read_character_info_payload()` 추가.
  - `load_character_info_by_endpoint()`은 레거시 호환 래퍼로 유지.
- 데이터 전달:
  - `data_json/_airflow_payloads/character_collect_<date>_<run_id>.json`

### 4) DAG 체인 재구성 + 적재 재시도
- 파일: `dags/maplemeta_dag.py`
- 변경:
  - 체인 구조를 아래와 같이 분리:
    - `ranker -> collect_ocid -> load_ocid -> collect_character_info -> load_character_info`
  - 공통 적재 재시도 함수 `_run_with_backoff()` 추가:
    - 지수 백오프: `10s -> 20s -> 40s ...` (최대 300초)
    - 적재 성공 전까지 재시도
  - `backfill_data()`에서 예외를 삼키지 않고 `raise` 하도록 변경.
- 효과:
  - 적재 실패가 성공으로 마킹되는 문제 방지.
  - 동일 API_KEY 내 다운스트림 오동작 차단.
