## 4. 업데이트(title) 파싱 규칙
### 4.1 version 파싱
- title 예: `클라이언트 1.2.411 업데이트 안내 (....)`
- 정규식: `(\d+)\.(\d+)\.(\d+)`
- 변환: `version_str = g1 + g2 + g3` → int
  - 예: 1,2,411 → "12411"
- 실패 시: version=NULL + warning 로그

### 4.2 content 파싱
- title의 괄호 `(...)` 내부 텍스트를 `content`로 저장
- 괄호 없으면 content=NULL

## 5. DW 적재 정책(핵심 설계)
### 5.1 업데이트 리스트는 notice_id 단위 row로 적재
- 동일 version에 대해 여러 notice_id가 존재 가능
- 따라서 `dw.dw_update`는 **notice_id PK/unique**로 upsert
- 컬럼(요구):
  - title, url, notice_id, date
  - version, content
  - detail_path
  - mahalil_path

### 5.2 detail_path 채움
- detail API는 notice_id당 1개이므로,
  - `detail_path = "/home/jamin/static/update/{version}_{notice_id}.html"`
- 파일 저장 성공 후 dw_update에 upsert/update로 반영

### 5.3 mahalil_path 채움(중요)
- 메할일은 version당 1개이므로,
  - `mahalil_path = "/home/jamin/static/update/{version}_mahalil.html"`
- 해당 version에 속한 **모든 dw_update row**에 동일 path를 채움
  - 구현 방식:
    1) 업데이트 리스트 적재 후 version 목록(distinct) 추출
    2) version별로 메할일 크롤링/저장 시도
    3) 성공하면 `UPDATE dw.dw_update SET mahalil_path=... WHERE version=...`
    4) 실패하면 NULL 유지 + warning 로그
