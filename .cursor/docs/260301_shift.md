# dm_shift_score / dm_balance_score DM 스키마 및 DW→DM ETL 명세

shift_score와 밸런스 점수(엔트로피)를 DM에 사전 적재한다. 대시보드는 조회만 수행한다.

---

## 1. DM 타깃 테이블 스키마

### 1.1 dm_shift_score

직업×세그먼트×버전별 Total Shift 점수. 랭킹은 `|total_shift|` 내림차순. **100점 척도** 컬럼은 상단 KPI 카드용.

| 컬럼명 | 타입 | NULL | 설명 |
|--------|------|------|------|
| version | VARCHAR(32) | N | 패치 버전 (예: 12409) |
| job | VARCHAR(64) | N | 직업명 |
| segment | VARCHAR(16) | N | 세그먼트: `50층`, `상위권` |
| outcome_shift | FLOAT | Y | Outcome Shift (무릉 참여·성과) raw |
| stat_shift | FLOAT | Y | Stat Shift (헥사/어빌/하이퍼) raw |
| build_shift | FLOAT | Y | Build Shift (스타포스/세트/무기) raw |
| total_shift | FLOAT | N | 가중 z-score 합산 (랭킹용) |
| direction | SMALLINT | Y | sign(total_shift): 1=상승, -1=하락, 0=변동없음 |
| outcome_score_100 | SMALLINT | Y | Outcome 0~100 척도 (KPI 카드용) |
| stat_score_100 | SMALLINT | Y | Stat 0~100 척도 (KPI 카드용) |
| build_score_100 | SMALLINT | Y | Build 0~100 척도 (KPI 카드용) |
| total_score_100 | SMALLINT | Y | Total 0~100 척도 (통합 KPI용) |

**100점 변환**: raw 절댓값을 버전 내 직업별 min-max 정규화 후 0~100 clamp. ETL에서 산출.

**PK**: (version, job, segment)

**세그먼트 정의**:
- `50층`: floor 50~69
- `상위권`: floor >= 90 (해당 버전 전체 상위권 n<15이면 floor >= 80)

**상단 KPI 카드 3개 분리**:
- **참여·성과**: outcome_shift / outcome_score_100 — 참여율, 50층 달성률, 최고 층수 등
- **스탯**: stat_shift / stat_score_100 — 헥사코어, 어빌리티, 하이퍼스탯
- **빌드**: build_shift / build_score_100 — 스타포스, 세트효과, 무기 등

---

### 1.2 dm_balance_score

버전×세그먼트별 밸런스 점수(정규화 엔트로피 기반). 직업 분포 균등도 0~100.

| 컬럼명 | 타입 | NULL | 설명 |
|--------|------|------|------|
| version | VARCHAR(32) | N | 패치 버전 |
| segment | VARCHAR(16) | N | 세그먼트: `50층`, `상위권`, `total` |
| balance_score | SMALLINT | N | 0~100 정수. 높을수록 균등 |
| top_job | VARCHAR(64) | Y | 1위 직업명 |
| top_share | FLOAT | Y | 1위 점유율 (0~1) |
| cr3 | FLOAT | Y | 상위 3개 직업 점유율 합 (0~1) |
| top_type | VARCHAR(32) | Y | (옵션) 1위 직업군 (전사/마법사/궁수/도적/해적) |
| top_type_share | FLOAT | Y | (옵션) 1위 직업군 점유율 |

**PK**: (version, segment)

**segment 값**:
- `50층`: mid (floor 50~69)
- `상위권`: top (floor >= 90, n<15이면 >= 80)
- `total`: α*50층 + (1-α)*상위권, 권장 α=0.7

## 2. DW 소스 테이블 가정

ETL 시 아래 테이블을 참조한다. 스키마는 `dw` 또는 `dm`(기존)을 사용할 수 있다.

### 2.1 dw_rank (또는 dm_rank)

무릉 랭크 raw/집계. shift_score·balance_score 공통 소스.

| 컬럼 | 타입 | 필수 | 용도 |
|------|------|------|------|
| version | VARCHAR | Y | 버전 필터 |
| date | DATE/TIMESTAMP | Y | 버전 내 날짜별 집계 |
| job | VARCHAR | Y | 직업명 (job_name 등) |
| job_type | VARCHAR | Y | 직업군 (전사/마법사/궁수/도적/해적) |
| floor | INT | Y | 최고층수 |
| record_sec | FLOAT | | 클리어 시간(초), 효율 계산용 |
| character_id | VARCHAR | | 캐릭터 단위 구분 (count용) |

**balance_score용**: job별 count, version+segment 필터  
**shift_score용**: share, clear50, topFloor, sec_per_floor 등 산출

### 2.2 dw_hexacore (또는 dm_hexacore)

헥사코어 pick/분포. Stat Shift용.

| 컬럼 | 타입 | 필수 |
|------|------|------|
| version | VARCHAR | Y |
| job | VARCHAR | Y |
| segment | VARCHAR | Y |
| hexa_core_name | VARCHAR | Y |
| hexa_core_type | VARCHAR | |
| count | INT | |
| total_level | INT | |

### 2.3 dw_ability, dw_hyper, dw_equipment

어빌리티/하이퍼/장비 분포. Stat·Build Shift용.

- job, version, segment, 항목별 count 또는 level
- JSD·TopKSwitch 계산에 사용

## 3. 계산 로직 요약

### 3.1 dm_balance_score

1. `dw_rank`에서 version·segment별 job별 count 집계
2. `p[j] = count[j] / total`, `H = -Σ p*ln(p)`, `E = H/ln(K)`
3. `balance_score = round(100 * E)` (0~100 clamp)
4. TopJob, TopShare, CR3, (옵션) TopType 산출
5. `total` 행: `round(0.7*50층 + 0.3*상위권)`

### 3.2 dm_shift_score

1. **Outcome**: share(로그비율), clear50, sec_per_floor 변화 → z-score 합산
2. **Stat**: 헥사/어빌/하이퍼 JSD + hexa_level_sum z
3. **Build**: 스타포스 z, weapon/set JSD, TopKSwitch
4. `total_shift = 가중*Z_O + 가중*Z_S + 가중*Z_B` (mid/top별 가중치 상이)
5. `direction = sign(total_shift)`
6. **100점 척도**: `*_score_100 = round(100 * (|raw| - min) / (max - min))` clamp 0~100. 버전 내 직업별 min/max 기준.

상세 공식: [260301_shift_score_집계.md](.cursor/docs/260301_shift_score_집계.md), [260301_entropy_집계.md](.cursor/docs/260301_entropy_집계.md)

## 4. DDL 예시

```sql
-- dm_shift_score (100점 척도 컬럼 포함)
CREATE TABLE IF NOT EXISTS dm.dm_shift_score (
    version VARCHAR(32) NOT NULL,
    job VARCHAR(64) NOT NULL,
    segment VARCHAR(16) NOT NULL,
    outcome_shift FLOAT,
    stat_shift FLOAT,
    build_shift FLOAT,
    total_shift FLOAT NOT NULL,
    direction SMALLINT,
    outcome_score_100 SMALLINT,
    stat_score_100 SMALLINT,
    build_score_100 SMALLINT,
    total_score_100 SMALLINT,
    PRIMARY KEY (version, job, segment)
);

-- dm_balance_score
CREATE TABLE IF NOT EXISTS dm.dm_balance_score (
    version VARCHAR(32) NOT NULL,
    segment VARCHAR(16) NOT NULL,
    balance_score SMALLINT NOT NULL,
    top_job VARCHAR(64),
    top_share FLOAT,
    cr3 FLOAT,
    top_type VARCHAR(32),
    top_type_share FLOAT,
    PRIMARY KEY (version, segment)
);
```

## 5. DW→DM ETL 작성 프롬프트

아래 프롬프트를 복사하여 ETL 쿼리 작성 도구(AI, DBA 등)에 전달한다.

```markdown
## 작업 요청: DW → DM ETL 쿼리 작성

### 목표
dw(또는 기존 dm) 소스 테이블에서 dm.dm_shift_score, dm.dm_balance_score 두 테이블로 데이터를 적재하는 SQL을 작성해주세요.

### 타깃 스키마
- dm.dm_shift_score: (version, job, segment, outcome_shift, stat_shift, build_shift, total_shift, direction, outcome_score_100, stat_score_100, build_score_100, total_score_100)
- dm.dm_balance_score: (version, segment, balance_score, top_job, top_share, cr3, top_type, top_type_share)

### 소스 가정
- dw_rank: version, date, job, job_type, floor, record_sec, (character_id 등)
- dw_hexacore: version, job, segment, hexa_core_name, hexa_core_type, count, total_level
- dw_ability, dw_hyper, dw_equipment: job, version, segment, 항목별 count/level

### 비즈니스 규칙

#### 세그먼트
- 50층: floor >= 50 AND floor <= 69
- 상위권: floor >= 90 (단, 해당 버전 전체에서 floor>=90인 행 수가 15 미만이면 floor >= 80)

#### dm_balance_score
1. 버전 v, 세그먼트 s별로 job_count[j] = count of rows (job별 캐릭터 수)
2. total = sum(job_count), p[j] = job_count[j]/total
3. H = -sum(p*ln(p)) for p>0, K = count of jobs with job_count>0
4. E = H/ln(K) (K<=1이면 E=0)
5. balance_score = round(100*E). clamp 0~100
6. top_job = argmax p, top_share = max p, cr3 = sum of top 3 p
7. top_type = job_type of top_job, top_type_share = sum(p where job_type=top_type)
8. segment='total' 행: balance_score = round(0.7*50층점수 + 0.3*상위권점수)

#### dm_shift_score
1. 버전 v-1(이전) vs v(현재) 비교
2. Outcome: share(로그비율), clear50, sec_per_floor 변화 → z-score 합산
3. Stat: 헥사/어빌/하이퍼 JSD + hexa_level_sum z
4. Build: 스타포스 z, weapon/set JSD, TopKSwitch
5. total_shift = mid/top별 가중치 적용 (mid: 0.5*O+0.3*S+0.2*B, top: 0.3*O+0.35*S+0.35*B)
6. direction = sign(total_shift)
7. 100점 척도: outcome/stat/build/total 각각 |raw|를 버전 내 직업별 min-max 정규화 후 round(100 * normalized) clamp 0~100

### 참조 문서
- 프로젝트 내 .cursor/docs/260301_shift_score_집계.md (shift_score 상세 공식)
- 프로젝트 내 .cursor/docs/260301_entropy_집계.md (밸런스 점수 상세)

### 출력 형식
1. dm_balance_score 적재용 INSERT ... SELECT (또는 MERGE)
2. dm_shift_score 적재용 INSERT ... SELECT (또는 MERGE)
3. 각 쿼리 상단에 주석으로 소스 테이블·컬럼 매핑 명시
4. 사용 DB 엔진: PostgreSQL
```
